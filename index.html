A propos de RocketMiX

RocketMiX est une plateforme de publication d'API, gratuite et open source, destinée aux entreprises qui souhaitent tourner leur SI vers l'extérieur le plus simplement possible !

En quelques mots, RocketMiX, c'est :

  • Une architecture orientées microservices
  • Une intégration ultra simplifiée et prépackagée des produits Netflix© Zuul, Eureka, Ribbon, Spring Boot Admin, et Swagger UI
  • Une installation réduite au strict minimum (juste un éxecutable Linux et rien d'autre! Pas de base de données, pas de broker, pas de messaging queue...)
  • Un portail utilisateur (avec génération automatique de la documentation), un bac à sable de test des API, une interface graphique de monitoring
  • Une configuration "zéro paramétrage"
  • Un module Spring Boot Starter pour vous aider à construire vos propres API
  • Une integration forte Linux pour vous permettre installer vos API en tant que service Linux (systemd)

Besoin d'une démo ? Testez RocketMix grâce à Heroku. (Identifiez-vous avec admin/admin)

5 minutes pour démarrer!

Vous avez une machine Linux 64 bits sous la main avec 2Go de Ram et 500Mo de disque sur disponible ? OK, vous êtes prêt!

En pré-requis, vous devez installer un JDK 8 ou supérieur. Faites
  • sudo yum install java-1.8.0-openjdk-devel sous CentOS
  • ou sudo apt install default-jdk sous Debian et Ubuntu

Téléchargez et lancez l'éxecutable Linux (une seule ligne de commande suffit!) :


wget https://sourceforge.net/projects/rocketmix/files/1.0.35/rocketmix-routing-server-1.0.35.war/download -O rocketmix-routing-server.war && chmod +x rocketmix-routing-server.war && export PID_FOLDER=$(pwd) && export LOG_FOLDER=$(pwd) && ./rocketmix-routing-server.war run

Le serveur API va démarrer sur le port 8080 (http://localhost:8080). Ouvrez un navigateur internet pour accéder à votre portail API. Il vous permet d'accéder au catalogue de vos API ainsi qu'à une interface de monitoring (le compte est admin/admin par défaut).

En production, vous pouvez installer votre serveur (et vos APIs) sous forme de service Linux. Lancez rocketmix-routing-server.war install pour déployer votre serveur en tant que service Linux (systemd). Respirez : systemd est sécurisé et vous garantie le redémarrage de votre serveur en cas d'arrêt. Et comme RocketMiX s'appuie sur des standards Linux, vous pouvez également contrôler votre serveur avec la commande systemctl status rocketmix-routing-server.

Développez vos propres API (en un temps record)

Vous êtes développeur Java et vous connaissez Spring et Maven ? Ca tombe bien, nous aussi!

On vous a préparé un Spring Boot Starter module pour simplifier le lancement de votre projet. Importez l'artifact suivant dans votre pom.xml :

<!-- https://mvnrepository.com/artifact/io.github.rocketmix/rocketmix-spring-boot-starter -->
<dependency>
    <groupId>io.github.rocketmix</groupId>
    <artifactId>rocketmix-spring-boot-starter</artifactId>
    <version>1.0.35</version>
</dependency>

A partir de là, le plus simple est d'aller consulter le projet de démo sur github.
