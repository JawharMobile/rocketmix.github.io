A propos de RocketMiX

RocketMiX est une plateforme de publication d'API, gratuite et open source, destinée aux entreprises qui souhaitent tourner leur SI vers l'extérieur le plus simplement possible !

En quelques mots, RocketMiX, c'est :

  • Une intégration ultra simplifiée et prépackagée des produits Netflix© Zuul, Eureka, Ribbon, Spring Boot Admin, et Swagger UI
  • Une installation réduite au stricte minimum (juste des éxecutables Linux et rien d'autre! Pas de base de données, de broker, de messaging queue...)
  • Un portail utilisateur (avec génération automatique de la documentation), un bac à sable de test des API, une interface graphique de monitoring
  • Une configuration visant le "zéro paramétrage"
  • Un Spring Boot Starter module pour vous faciliter la construction de vos propres API
  • Un generateur de scripts Linux pour installer vos API en tant que service systemd
  • Une architecture orientées microservices

5 minutes pour démarrer!

Vous avez une machine Linux 64 bits sous la main avec 2Go de Ram et 500Mo de disque sur disponible ? OK, vous êtes prêt!

Vous allez copier et lancer 3 éxecutables nécessaires à faire fonctionner RocketMiX :
  • Le routeur (rocketmix-routing-server) permettant de recevoir tous les appels aux API. Ce routeur fait aussi office de portail API
  • Le gestionnaire (rocketmix-management-server) qui est en charge de savoir quelles API sont disponibles
  • Une API HelloWorld (rocketmix-services-demo) de démonstration
Pour des questions de simplicité, nous allons tout éxecuter sur la même machine. Dans la pratique, nous vous conseillons de répartir vos éxecutables sur différentes machines.


Etape 1 : Téléchargez les éxecutables

En pré-requis, vous devez installer un JDK 8 ou supérieur. Faites
  • sudo yum install java-1.8.0-openjdk-devel sous CentOS
  • ou sudo apt install default-jdk sous Debian et Ubuntu
Téléchargez les exécutables (les fichiers avec une extension .war) sur sourceforge :



Ajoutez les droits d'executions sur ces fichiers : 

chmod +x rocketmix-routing-server-xxx.war            
chmod +x rocketmix-management-server-xxx.war            
chmod +x rocketmix-services-demo-xxx.war

Etape 2 : Lancez le routeur, le manager et l'API de démo

Lancez (dans l'ordre que vous voulez!) les différents éxecutables que vous venez de télécharger dans des terminaux différents : 

./rocketmix-routing-server-xxx.war            
./rocketmix-management-server-xxx.war            
./rocketmix-services-demo-xxx.war
Le routeur va démarrer sur le port 8080 ( http://localhost:8080). Le gestionnaire sur le port 8761 (http://localhost:8761/admin). La démo sur le port 8083 (http://localhost:8083).

Ouvrez un navigateur internet et allez sur http://localhost:8080 afin d'accéder à votre portail API. Il vous permet d'accéder au catalogue de vos API ainsi qu'à une interface de monitoring (le compte est admin/admin par défaut).

En production, pas de terminal! Vous pouvez installer tout cela sous forme de services Linux. Lancez les exécutables avec l'option --install. Cela vous génèrera l'ensembles des scripts et fichiers d'installation en tant que service systemd. Pour personnaliser davantage votre portail, utilisez l'option --help pour connaitre l'ensemble des paramètres possibles. Pour en savoir plus, vous pouvez consulter la documentation officielle de Spring.

Etape 3 : Développez vos propres API

Vous êtes développeur Java et vous connaissez Spring et Maven ? Ca tombe bien, nous aussi! Du coup, on vous a préparé un Spring Boot Starter module pour simplifier le lancement de votre projet. Importez l'artifact suivant dans votre pom.xml : 

<!-- https://mvnrepository.com/artifact/io.github.rocketmix/rocketmix-spring-boot-starter -->
<dependency>
    <groupId>io.github.rocketmix</groupId>
    <artifactId>rocketmix-spring-boot-starter</artifactId>
    <version>1.0.23</version>
</dependency>
A partir de là, le plus simple est d'aller consulter le projet de démo et la page github du projet ici.
