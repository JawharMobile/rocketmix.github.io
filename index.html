A propos de RocketMiX

RocketMiX est une plateforme de publication d'API, gratuite et open source, destinée aux entreprises qui souhaitent tourner leur SI vers l'extérieur le plus simplement possible ! En quelques mots, RocketMiX, c'est :

  • Une intégration ultra simple et prépackagée des produits Netflix© Zuul, Eureka, Spring Boot Admin, et Swagger UI
  • Une installation réduite au stricte minimum (juste des éxecutables Linux et rien d'autre! Pas de base de données, de broker, de messaging queue...)
  • Un portail utilisateur (avec génération automatique de la documentation), un bac à sable de test des API, une interface graphique de monitoring
  • Une configuration visant le "zéro paramétrage"
  • Un template Spring Boot prêt à l'emploi pour construire vos propres API
  • Une architecture orientées microservices

5 minutes pour démarrer!

Vous avez une machine Linux 64 bits sous la main avec 2Go de Ram et 500Mo de disque sur disponible ? OK, vous êtes prêt!



Vous vous apprêter à copier et à lancer 3 éxecutables nécessaires à faire fonctionner RocketMiX :
  • Le routeur permettant de recevoir tous les appels aux API. Ce routeur fait aussi office de portail API
  • Le manager qui est en charge de savoir quelles API sont disponibles
  • Une API HelloWorld de démonstration
Pour des questions de simplicité, nous allons tout éxecuter sur la même machine. Dans la pratique, nous vous conseillons de répartir vos éxecutables sur différentes machines.


Etape 1 : Téléchargez les éxecutables

En pré-requis, vous devez installer un JDK 8 ou supérieur. Faites
  • sudo yum install java-1.8.0-openjdk-devel sous CentOS
  • ou sudo apt install default-jdk sous Debian et Ubuntu
Vous pouvez récupérer les executables directement ici ou bien lancer la commande suivante : wget ... & wget ... & wget ....

Ajoutez les droits d'executions sur ces fichiers chmod +x microservices*



Etape 2 : Lancez le routeur, le manager et l'API de démo



Dans l'ordre que vous voulez, lancez successivement les différents éxecutables que vous venez de télécharger. Les logs vont sortir à la console; du coup, nous vous conseillons de les lancer dans des terminaux différents. Le routeur va démarrer sur le port 8080. Le manager sur le port 8761. La démo sur le port 8081. Toutes ces briques répondent en http sur leurs ports respectifs.

Ouvrez un navigateur internet et allez sur http://localhost:8080 afin d'accéder à votre portail API!

En production, pas de terminal! Vous pouvez installer tout cela sous forme de services Linux. Vous trouverez des exemples sur le repo github du projet. Vous pouvez aussi vous appuyez sur la documentation officielle de Spring.
